flex意为“弹性布局”，为盒装模型提供最大的灵活性，任何一个元素都可以指定为flex布局

flex 布局的可用属性

  • flex-direction 主轴排列方向
  • flex-wrap 主轴不够是否换行
  • flex-flow 是前面两个的简写
  • justify-content 项目在水平方向上的对齐方式
  • align-items 在同一行中的项目在垂直方向上的对齐方式
  • align-content 当有多行的时候 行在垂直方向上的对齐方式

justify-content属性定义元素在主轴上的对齐方式 flex-start/flex-end/center/space-between/space-around
比较特殊的是space-around 和space-between 前者每个项目之间的间隔相等，后者 两端的元素对齐主轴的开始和结束，项目之间的间隔相等

align-items用来在交叉轴上对齐元素，比如在一行元素内，各个元素高度不统一，那么这一行内的元素该怎么对齐

  • flex-start：交叉轴的起点对齐。
  • flex-end：交叉轴的终点对齐。
  • center：交叉轴的中点对齐。
  • baseline: 项目的第一行文字的基线对齐。
  • stretch（默认值）：如果项目未设置高度或设为auto，将占满整个容器的高度。

align-content属性定义了多根轴线的对齐方式。如果项目只有一根轴线，该属性不起作用。也就是当主轴方向上有多行内容的时候
这些行在交叉轴方向上该怎么排列。共有六种属性值，可以看到和justify-content属性值高度平行。
切记保证垂直轴上有剩余的空间来布局，否则会没有效果

  • flex-start：与交叉轴的起点对齐。
  • flex-end：与交叉轴的终点对齐。
  • center：与交叉轴的中点对齐。
  • space-between：与交叉轴两端对齐，轴线之间的间隔平均分布。
  • space-around：每根轴线两侧的间隔都相等。所以，轴线之间的间隔比轴线与边框的间隔大一倍。
  • stretch（默认值）：轴线占满整个交叉轴。

flex 项目上的可用属性

  • order 指定项目的显示顺序
  • flex-grow 有剩余空间时，项目占用剩余空间的份数，为0表示不变
  • flex-shrink 空间不足时候，各个项目缩小 为0表示空间不足也不缩小
  • flex-basis 暂时不知道是个咋回事
  • flex 上面三个属性的缩写，grow shrink basis
  • align-self 项目可以覆盖掉 align-items的属性值，自己有自己的在垂直方向上的对齐方式

flex-shrink，空间不足时候 缩小

计算规则： 按照shrink的比例，消耗容器总超出的空间，本例中，每个项目宽度50px，flex宽度110，所以超出了 3*50-110=40px,第三个项目shrink为0，所以不会被缩小，那么项目1和2将吸收掉这些超出空间，也就是40px，项目1 shrink为2，项目2 shrink为1 那么项目1的宽度为 50-40*2/3=23.33 项目2 的宽度为 50-40*1/3=36.67
